{
카메라 플러그인
- 기본적으로 유니티와 연결이 될 것임
- 추후에는 플러그인 형식으로, 동적 라이브러리 형식으로 프로젝트에 합쳐질 예정
- 이미지를 통한 물체 감지 기능
- 물체 감지는 OpenCV 라이브러리를 이용한 이미지 분석 방식
}

{
카메라 모듈의 의도
- 카메라를 통해 골프공 등의 둥근 공 형태를 감지
- 감지된 공의 위치
- 감지된 공에 번호를 붙이고 추적하여 해당 공의 속도와 방향 계산
- 계산된 값을 Manager 등의 객체에 지속적으로 갱신 (클라이언트에서 필요할 때 꺼내갈 수 있도록)
- 2개의 카메라를 통해 수직과 수평으로 구하여 각 축에 대한 정보를 얻을 수 있도록 함
- 두 카메라에서 감지 중인 공의 동기화 기능 (같은 공을 추적하도록)
- 카메라 렌즈의 왜곡에 따른 값의 변형을 최소화 할 수 있는 로직 내재
- 공이 시야에서 사라질 경우 마지막 값으로 갱신을 멈추고 현재 추적되지 않는다는 flag 상태 값 표시
}

{
카메라 모듈의 동작
- 함수를 통한 모듈 기동 (StartCameraModule 등의 함수를 외부에서 호출, 기동 성공 유무에 따른 콜백 제공)
- 함수를 통한 모듈 중지 (중지 성공 유무에 따른 콜백 제공)
- 현재 기동 상태 변수 제공
- 중지 전까지는 루프를 돌며 계속 동작
- 기기에 연결되어있는 카메라를 감지하여 사용할 것
- 연결되는 카메라의 개수는 두 개
	-> 연결되는 카메라 두 개는 각각 구분하여 원하는 곳에 할당이 가능하도록 (구분이 되어야하고, 그 설정을 사용자 의도대로 구분할 수 있게. 방법 제시 부탁)
- 찾지 못한 카메라의 경우 에러 반환 예외처리와 함께 알 수 있도록 작동 (연결이 된 카메라는 정상 작동으로 처리)
- 각 카메라는 스레드를 이용하여 2 종류의 카메라에서 분석한 정보를 따로 갱신
- 카메라 이미지 해상도, 목표 fps 등은 추후에 쉽게 변경 할 수 있도록 작성 (클라이언트에서 실시간으로 변경 가능하도록 함수 등을 제공하면 좋을 듯)
- 카메라는 추상화하여 작성
}

{
카메라 모듈 동작 단계
- 모듈 전체 관리 부분
- 카메라 관리 부분
- 카메라 인식 동작 부분
- 카메라 이미지 떠오는 동작 부분
- 이미지 분석 부분
- 데이터 갱신 부분
다음과 같은 부분으로 나누어 짜여지도록 하고, 각 기능은 필요에 따라 가져와 사용할 수 있는 형식으로 함수 및 클래스를 독립적으로 작성
}

{
모듈의 큰 구조
- 동작부
- 로직 및 클래스 구현부
- 데이터 저장부
- 인터페이스 (외부 기능 노출)

아키텍처는 MVP 패턴을 따르도록 하여 작성
}
